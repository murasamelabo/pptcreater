Add content writing review framework

Add a content-review framework that prevents AI-generated slides from reading
like long documents. The new core rules are locale-aware and content-mode-aware:
Japanese report, technical, and handout decks use a short topic-label title plus
a separate slide message; Japanese presentation and decision decks allow concise
assertion titles; English decks use action-title guidance with answer-first
supporting evidence.

Expose the same framework through:
- core reviewDeckContent/getContentGuidance APIs
- lint_deck warnings and suggestions for generic titles, verbose titles,
  missing Japanese slide messages, long messages, prose-like body text, and too
  many bullets
- MCP review_content tool
- CLI content-review command
- skill packs, installed project guidance, README, and MCP resources/prompts

Regenerate overview sample DeckSpecs/PPTX/Studio HTML so the public examples
follow the new content framework.

Verified with build, full test suite, content/error lint check for sample decks,
and code review.
